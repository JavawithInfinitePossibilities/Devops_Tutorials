    18. Fetch vs Pull
        Git Fetch: The git fetch command is used to retrieve the latest updates (commits, files, and references) from the remote repository and store them in your local Git repository. 
        1.  Does not modify your local working directory or current branch.
        2.  Updates remote-tracking branches (e.g., origin/main), providing a way to see what others have done without disrupting your local work.
        3.  Allows you to review or compare changes before integrating them using commands like git log or git diff.
        Git Pull: The git pull command automates the process of synchronizing your local branch with its remote counterpart. It is essentially two operations in one: git fetch followed by either a git merge or git rebase (depending on configuration).
        1.  Downloads and immediately integrates the remote changes into your current local branch.
        2.  Can cause merge conflicts if the remote changes conflict with local modifications, pausing your workflow until resolved.
        3.  Updates your working directory files to match the latest changes from the remote.
